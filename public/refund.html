환불정책

시행일: 2026년 5월 1일

1. 기본 원칙

크레딧은 보고서 초안 생성 기능 이용을 위한 선불 이용권입니다. 생성 요청 전에는 사용자가 생성을 취소할 수 있으며, 생성이 완료되지 않은 실패 작업에는 원칙적으로 크레딧을 차감하지 않습니다.

생성된 보고서에 내용상 오류가 하나라도 확인되는 경우 사용자는 환불 또는 해당 보고서 크레딧 복구를 요청할 수 있습니다. 이 정책은 AI 결과물의 한계를 고려해 사용자를 더 두텁게 보호하기 위한 기준입니다.

2. 환불 가능 사례

  • 서비스 오류로 보고서 파일이 생성되지 않았는데 크레딧이 차감된 경우
  • 중복 차감, 관리자 입력 실수, 충전 오류가 확인된 경우
  • 보고서에 사실 관계 오류, 계산 오류, 화학·물리 개념 오류, 실험 절차 오류, 출처·인용 오류, 내용 누락 등 내용상 오류가 하나라도 확인된 경우
  • 생성 파일이 열리지 않거나 형식 문제로 내용을 정상 확인할 수 없는 경우
  • 사용하지 않은 크레딧에 대해 운영자가 환불 가능하다고 인정한 경우

3. 환불이 제한될 수 있는 사례

  • 내용상 오류 없이 문체, 어투, 디자인, 분량 취향, 평가 만족도 등 주관적 사유만 있는 경우
  • 사용자가 잘못된 파일을 업로드했거나, 부정확한 입력값을 제공했거나, 학교 규정에 맞지 않게 사용한 경우
  • 금지 행위, 계정 공유, 권한 없는 자료 업로드가 확인된 경우
  • 동일 생성 건에 대해 이미 환불 또는 크레딧 복구가 완료된 경우

단, 보고서 내용 자체의 오류가 확인되는 경우에는 위 사유와 별개로 해당 생성 건의 환불 또는 크레딧 복구를 검토합니다. 학칙 위반, 저작권 침해, 제출 후 평가·징계 등 사용 방식에서 발생한 손해는 환불 대상 손해로 보지 않습니다.

4. 환불 요청 방법

환불 요청 시 사용자 이름, 생성 시각, 보고서 종류, 오류가 있는 문장·표·수식·출처 위치, 오류 설명, 결제·충전 내역을 계정 발급자 또는 서비스 운영자에게 알려주세요. 가능하면 생성 파일 또는 오류 부분 캡처를 함께 보내주세요.

운영자는 사용 로그, 생성 파일, 사용자가 제시한 오류 내용을 확인해 환불 또는 크레딧 복구 여부를 안내합니다. 내용상 오류가 확인되면 다운로드 여부와 관계없이 환불 또는 크레딧 복구 대상이 됩니다.

5. 청약철회와 유료 판매 고지

현재 크레딧은 제한된 사용자에게 관리자 수동 발급 방식으로 운영되며, 공개 자동 결제나 불특정 다수 대상 판매 기능을 제공하지 않습니다. 유료 크레딧을 반복적으로 판매하거나 공개적으로 모집하는 경우 전자상거래 관련 법령상 사업자 정보 표시, 통신판매업 신고, 청약철회, 세금 처리 등 별도 의무가 발생할 수 있습니다.

정식 유료 판매 전에는 판매자 정보, 연락처, 가격, 결제 방법, 청약철회·환불 기준, 소비자 피해 처리 방법을 별도 고지합니다.
